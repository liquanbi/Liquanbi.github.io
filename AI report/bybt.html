为什么品牌方需要这套系统？

防范价格破发

百亿补贴是乱价重灾区。官方补贴叠加店铺暗降，极易导致最终到手价（单件补贴价）击穿品牌底价，引发全渠道价格体系崩盘。

追溯乱价源头

部分经销商利用京东/拼多多的平台展示机制，隐藏实际店铺名称进行低价抛售。系统能剥离前端伪装，还原真实的供货责任方。

洞察活动商机趋势

分析每期百亿补贴活动中，品牌各SKU的参与规模（参与商品数/店铺数）。以此评估经销商的参与热度与渠道铺货方向，辅助营销决策。

价格监测基准
· 补贴价 (总价)
· 件数 (如: 60粒*2瓶=2件)
· 单件补贴价
 直接计算出最终到手均价。排查组合装（如买2发3）的隐藏破发行为，这是防乱价的最核心判定指标。
渠道与责任主体
· 平台 (京东/拼多多/淘宝)
· 店铺名称
· 店铺展示标签 (shop_flag)
· 宝贝链接
 锁定违规源头。配合 店铺展示标签 识别前台隐藏名称的店铺，提供带有唯一标识的（链接）供品牌方进行渠道扣罚。
产品字典映射
· 宝贝名称 / 属性名称 (原值)
· 目录名称 (标准化)
· code / sku
 应对经销商店铺五花八门的商品命名。通过算法将非标名称清洗映射为品牌方的标准 Code，便于内部拉取报表对账。
活动商机分析
· 日期
· 参与活动记录明细
 通过累加计算各周期的【参与百补的商品总数】，分析不同品类、不同经销商在商机活动中的活跃度，指引渠道铺货方向。
系统执行流程

从“原始网页”到“业务决策数据”

提供全链路的数据清洗与结构化处理能力。

1. 全景原值抓取

获取: 宝贝名称、原始属性
例: "Swisse升级款7000MG痛风片... 60粒*两瓶"

2. SKU与件数拆解清洗

提取件数: "两瓶" -> 件数=2
计算均价: 补贴价/件数 -> 单件补贴价

3. 品牌字典标准映射

映射目录: Swisse二代西芹籽精华片
输出 Code: 01-F10-10004

最终输出为标准 CSV/Excel 报表，或通过 API 直传品牌方 BI 系统，实现无缝对接。

构建全渠道业务数据闭环

基于三大核心维度，助力品牌方在百亿补贴渠道中实现精细化运营：
通过【单件补贴价】死守价格底线，
通过【店铺展示标签】追溯违规主体，
通过【活动参与规模】洞察渠道商机。

系统最终交付形态：

以标准化数据资产（Data Assets）的形式，直接对接企业内部BI系统或以自动化报表输出。彻底解决人工巡查时面临的“算不清满减均价”、“找不准供货店铺”、“对不齐内部Code”三大难题，全面提升品牌对下游渠道的管理能效。
